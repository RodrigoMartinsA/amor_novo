📊 Testar Performance

1. Google Lighthouse

1. Abra o site no Chrome
2. Pressione F12
3. Clique em "Lighthouse" (ou "Audits")
4. Clique "Analyze page load"
5. Aguarde o resultado

Metas para este site:
- Performance: 90+
- SEO: 95+
- Acessibilidade: 90+

2. PageSpeed Insights

1. Vá em pagespeed.web.dev
2. Cole a URL do seu site
3. Analise o resultado

Importante: Este site deve ter notas altas!

3. Testar no Seu PC

Abra o DevTools (F12) > Network:

- Tamanho total deveria ser < 1MB
- Carregamento < 2 segundos em conexão 4G
- Teste com "Slow 4G" ou "Offline"
- Verifique "Largest Contentful Paint" (LCP)

🌐 Opções de Deploy (Recomendadas)

1. Netlify (MAIS FÁCIL ⭐)

Passos:
1. Vá em netlify.com
2. Clique "Drop files here to deploy"
3. Arraste a pasta do seu projeto
4. ✨ Pronto! Seu site está online

Resultado:
- URL gratuita: seu-projeto-1234.netlify.app
- SSL/HTTPS automático
- CDN global (rápido em qualquer lugar)
- Sem configuração necessária

2. Vercel

Passos:
1. Crie conta em vercel.com
2. Clique "Import Project"
3. Selecione a pasta
4. Deploy automático

Resultado:
- URL similar: seu-projeto.vercel.app
- Deployment automático via Git
- Analytics integrado

3. GitHub Pages (Para Devs Git)

Passos:
1. Crie repositório em GitHub
2. Clique em Settings > Pages
3. Selecione "main" branch como source
4. Aguarde 1-2 minutos

Resultado:
- URL: seu-usuario.github.io/nome-repo
- Sempre atualizado com seus commits
- Gratuito e profissional

4. Seu Próprio Hosting (Se já tem)

Passos:
1. Faça upload dos arquivos via FTP/SFTP
2. Certifique-se que index.html está na raiz
3. Configure SSL/HTTPS (muito importante!)

Dica: Use Let's Encrypt (gratuito)

🔒 Segurança & Boas Práticas

Que Já Está Implementado:

  • ✅ Sem banco de dados (não há dados para roubar)
  • ✅ Sem logins/senhas
  • ✅ Sem requisições AJAX (seguro contra XSS)
  • ✅ HTML semântico (busca melhor no Google)
  • ✅ HTTPS ready (funciona com SSL/TLS)
  • ✅ CSP headers compatível (sem inline scripts)

Adicional (Opcional):

Se adicionar formulário de contato:

❌ NÃO FAÇA:
- Enviar dados para emails diretos via JS
- Guardar senhas no código
- Usar HTTP (sempre use HTTPS)

✅ FAÇA:
- Use serviço como Formspree, Basin ou Netlify Forms
- Valide campos no cliente E no servidor
- Use HTTPS sempre
- Sanitize inputs

Exemplo com Formspree (grátis):
<form action="https://formspree.io/f/YOUR_ID" method="POST">
  <input type="email" name="email" required>
  <textarea name="mensagem"></textarea>
  <button type="submit">Enviar</button>
</form>

Melhorar SEO (Opcional):

1. Adicione meta description no <head>:

2. Adicione Open Graph (para redes sociais):

<meta property="og:image" content="https://seu-site.com/preview.jpg">

<link rel="icon" href="favicon.ico">

4. Sitemap (para Google indexar):
Criar arquivo sitemap.xml:

<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>https://seu-site.com</loc>
    <lastmod>2025-04-19</lastmod>
  </url>
</urlset>

⚙️ Otimizações Avançadas (Se quiser)

Gzip Compression

Se usando Netlify/Vercel: Automático! ✨

Se seu hosting suporta:
Coloque .htaccess:
<IfModule mod_deflate.c>
  AddOutputFilterByType DEFLATE text/html text/plain text/xml
  AddOutputFilterByType DEFLATE text/css
  AddOutputFilterByType DEFLATE application/javascript
</IfModule>

Caching

Se usando Netlify/Vercel: Automático!

Coloque no _redirects (Netlify):
[[headers]]
  for = "/index.html"
  [headers.values]
    Cache-Control = "max-age=3600"

  for = "/styles.css"
  [headers.values]
    Cache-Control = "max-age=31536000"

🐛 Debug & Troubleshooting

Console (F12 > Console):

Se ver erros vermelhos:
- Clique no erro para ver a linha
- Verifique se arquivos existem (404)
- Verifique caminhos das imagens

Script típico para debug:
console.log('🎨 Portfólio carregado'); // Vê se JS rodou

Network (F12 > Network):

Verifique que tudo carrega:
- status 200 = OK
- status 404 = arquivo não encontrado
- status 500 = erro do servidor

Tamanho esperado:
- HTML: ~5KB
- CSS: ~17KB  
- JS: ~5KB
- Imagens: varia (deve ser comprimido!)
